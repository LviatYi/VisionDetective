# Vision Detective 视界侦探

**Vision Detective 视界侦探** 是一款桌面解密游戏，玩家需要在由卡牌构筑的层层谜题中拓展视界，寻找答案。

风格关键字：纯黑色简笔画 福尔摩斯时代 英国伦敦 古典欧式 线条圆润

- 正交垂直俯视角，2D，通过 2D 模拟的 3D 动画。

## 技术栈选型

- Rust Bevy
- AI 制作素材。

## 要素

### 硬币

表示角色单位。

#### 主角

一个带有侦探头像的硬币，代表玩家控制的角色。

通过某种表现形式，可以将硬币 “弹开” 以移动。把硬币弹到卡牌上进行交互。

玩家具有视野，是一个以硬币为中心的圆形区域。

#### NPC

#### 敌对角色

一个带有特殊任务头像的硬币，表示敌对角色。它们会朝玩家弹出攻击性道具，或是朝玩家翻滚，玩家需要躲避它们。

#### 道具

一个带有图像的硬币，表示一次性物品，玩家可以通过角色可交互卡牌拾取它们，并在后续事件中使用他们。

使用方式类似于角色单位的弹开。

---

### 卡牌

游戏中任何非角色单位都以卡牌形式存在。卡牌平铺在 2D 场景上。

> 卡牌资产应该由简笔画风格的图像绘制，具有清晰的线条和简洁的设计。
> 资产作为可导入素材，其背景应该是透明的。

卡片由名称 + 图像 + 背景组成。

卡牌具有不同类型。类型则带有标签，以表达某种共性。

- **承载** 玩家将硬币弹到它上面，可以随之移动，并越过障碍。

**照明**，**视野条件**： 玩家静止时，视野范围盖住可交互卡牌的区域将被「照明」。当照明区域占该卡牌一定比例时，玩家就能看到该卡牌。

#### <div style="color:#A4ACC6;">类型: 障碍 ■ #A4ACC6</div>

表达视界限制的核心概念。

意味着该卡片除了名称和图像之外，还包含骨骼。它是一段闭合路径，玩家的视野将被其阻挡。

同时，角色硬币无法叠在障碍卡牌上，角色撞到障碍卡牌时会弹开。

**弹开**： 弹开是一种物理模拟机制，存在的意义在于对玩家的控制造成一定难度。

> example: 墙 树木 椅子 桌子

#### <div style="color:#FFD20D;">类型: 角色可交互 ■ #FFD20D</div>

表示角色可以与该卡牌进行交互。

玩家可以通过将硬币弹到该卡牌上来触发交互事件。

> example: 对话事件 获取道具事件

#### <div style="color:#3F2111;">类型: 线索 ■ #A4ACC6</div>

> - **好奇** 玩家需要达成视野条件才能看到它的详细信息，否则显示问号。

线索下方叠了一张可交互卡牌。达成视野条件时，下方的可交互卡牌将自动滑出来。

#### <div style="color:#E0E0E0;">类型: 景观 ■ #E0E0E0</div>

无影响物品，仅作为装饰存在。某些可交互物在交互完成后会变成景观。

#### <div style="color:#590163;">类型: 道具可交互 ■ #590163</div>

表示道具可以与该卡牌进行交互。

**照明**，**视野条件**： 玩家静止时，视野范围盖住可交互卡牌的区域将被「照明」。当照明区域占该卡牌一定比例时，玩家就能看到该卡牌。

玩家可以通过将硬币弹到该卡牌上来触发交互事件。

### 关卡

#### DEMO 关（第一关） 5-10 min

[关卡设计](./demo-level.drawio)

可玩点：

- 玩家需要将自己弹到合适的位置，才能看到视野条件卡牌。
- 玩家需要将自己弹到（方向控制 + QTE？）可交互卡牌上来触发事件。有的可交互卡牌由于与障碍重叠，意味着弹上去有一定难度。
- 暂定为线性解密。玩家在对话时做选择题，来推动剧情发展并达到结局。

剧情线：

- 玩家在深夜街边发现了一个惊慌失措的人，自称刚刚听到楼下邻居家里发生了尖叫。玩家决定帮助他调查这个案件。（这个邻居即这起盗窃谋杀案的凶手）
    - 全出场 NPC 仅 1 人。
- 房间 A 发现尸体。还有血迹作为线索。
    - 场景结构要简单，地图不能太大。
    - 滴落的血迹作为线索，引导玩家去房间 B。**滴落血迹在形态上具有运动学特征**。
    - 线索被遮挡，需要玩家深入到某个位置才能看到。（核心扣题机制）
- 房间 B 的血迹指明第一现场，发现了一个被盗的保险箱。阳台门虚掩。
- 阳台可见楼层不高，翻出即可到达多路口深巷。**但整条栏杆积灰完整**，提示凶手没有从阳台逃走。
- 阳台另一侧有逃生梯，门虚掩，可以通向楼上。**其它邻居似乎没有注意到异常**。
- 确定真凶。

剧情线自洽性意见：

- 没有明确证据。（可以通过压力让真凶自己承认？）
- 真凶如何出现在楼下？假如顺着逃生梯逃跑，应该回家了。

#### 第二关 15-30 min

可玩点：

- 深入照明机制，有的条件藏在由带缝隙的障碍物构成的不可达区域里。

## 功能模块

### 物理模拟

物理模拟的功能目标旨在提供物理运算机制，以实现硬币（角色）弹射、掉落、触地等物理效果。

物理模拟机制采样自现实世界中的物理现象，并进行风格化调整。

#### 硬币运动轨迹

在 VD 中，角色控制的硬币可能被弹起，并进行一段模拟的抛物线运动。

在 z 轴上：

- 硬币首先被瞬间给予向上的初速度 $v_0$，该大小由玩家输入影响，处于一个范围内。
- 随后受到重力加速度 $g$ 的影响。
- 在掉落地面的瞬间，硬币在 z 轴上的速度将取反，并乘以一个小于 1 的速度损失系数（模拟弹性系数 $e$），以模拟弹起效果。
- 落地次数 $c_g$ 达到 2 次时，z 分量的速度将被设置为 0，硬币将停止弹起并开始滑行。

$$
V_z =\begin{cases}
v_0 - g t && \text{if } t \lt 2 \frac{v_0}{g} \\
v_0  e && \text{if } 2 \frac{v_0}{g} \le t \le 2 ( \frac{v_0}{g} + \frac{v_0 e}{g} ) \\
0 && \text{if } t > 2 ( \frac{v_0}{g} + \frac{v_0 e}{g} )
\end{cases}
$$

在 x 和 y 轴上：

- 硬币首先被瞬间给予平面初速度 $v_0$，该大小由玩家输入控制，处于一个范围内。
- 不受空气阻力影响，在触地时按速度损失系数 $e$ 立即损失一定比例的速度。
- 在滑行阶段中，硬币受到摩擦系数 $f$ 影响，摩擦系数非常大，能够使其快速停止。

#### 障碍物系统

卡牌可能包含障碍物，当角色硬币与障碍物发生碰撞时，硬币会弹开。

障碍物由闭合路径组成，闭合路径可能包含多个子路径，子路径允许是直线或贝塞尔曲线。

### 角色

#### 输入控制

玩家主要通过鼠标进行控制。鼠标悬浮到角色硬币上时，显示一个图标（手指弹硬币的样子）表示发力点，长按进行 QTE，松开后角色弹出。

用硬币图形大小模拟被弹起的高度。

存在一个 InputLocker，外部可通过它，传入一个字符串，以锁定禁用用户输入。

#### 视野机制

玩家具有一个以角色为中心，固定值为半径的圆形视野区域。

视野区域会被障碍物遮挡。

通过迷雾（半透明的黑色遮罩）来表现视野遮挡的效果。

### 对话交互模块

玩家可以通过与特定卡牌的交互触发对话事件。

在表现上，对话交互将展示一个对话框。主角的立绘将出现在左侧，而右侧则展示对话目标的立绘。

- 对话框是一个半透明的黑色矩形，位于屏幕底部。具有合适的高度，宽度则与屏幕一致。
- 对话框内展示对话来源的名字和对话文本。
- 点击对话框或按下空格键可以推进对话。
- 当对话来源正在说话时，立绘才会显示。

在配置上，交互对话由一系列节点构成。节点将包含：

- ID
- 对话来源
- 对话文本

首个节点视为对话头。

### 镜头控制

一般地，地图大于摄像机视口。当玩家操控硬币移动时，摄像机将跟随硬币移动，以保证硬币仍处于镜头内。

**视口边缘** 窗口的上下左右边缘（各占边长的 20%）构成视口边缘区域。

镜头控制有几个阶段：

- HoldOn 当硬币未运动或硬币运动且远离视口边缘时，摄像机保持静止。
- Pursue 当硬币运动且触碰视口边缘时，摄像机立即以特定加速度跟随，其中加速度值使摄像机能在硬币以当前速度继续移动固定距离时追上硬币速度。
- Follow 追上硬币速度后，硬币与摄像机保持相对静止，即便硬币因碰撞发生速度跳变。
- Focus 当硬币运动结束后，摄像机将平滑移动以使硬币回到视口中心。

当镜头处于 Focus 时，角色不接受输入。

### 场景编辑器

应有某种场景编辑器，方便设计关卡。

编辑器界面包含

- 主要场景编辑区
    - 显示正在编辑的场景
    - 备忘录模式，可以撤销 (Ctrl Z) 或重做 (Ctrl Shift Z)
    - 可以选中卡牌，随后进行针对性编辑。可以长按拖动来改变位置，可以长按卡牌外角拖曳点来改变旋转。可以按下 delete 进行删除。
- 左侧卡牌预制体区
    - 可以将卡牌预制体拖到场景编辑区中
- 能够以某种格式导出场景数据。应同时支持轻量二进制与可读两种格式。
- 能够以某种格式导入场景数据，并在编辑器中显示。
